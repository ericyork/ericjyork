How to Set up 11ty, AlpineJS, and Tailwind CSS using Vite

Eleventy, Alpine, Tailwind and Vite are bleeding edge tools for enabling rapid development using JS. This stack allows you to convert markdown to html, add javascript and css to the render, and bundle it all up to deploy at the smallest sizes possible. However, getting all of them to work together nicely is no mean feat, so that’s what this guide is for.

By way of introduction, Eleventy is a static site generator that allows structuring and templating content using a variety of templating languages, AlpineJS is a lightweight library for UI enhancement, Tailwind CSS is the utility-first stylesheet framework, and Vite is a very fast bundler and development server. Together, they make a powerful full stack JAM stack.

Prerequisites

To follow this guide, you’ll need a good IDE (like VSCode), node and npm, basic knowledge of markdown, HTML, CSS, and JS, and the ability to work from a Linux-like command line.

1. Initialize Vite

First things first, find a good folder to work in, such as your desktop or documents folder, and navigate there in your terminal. Initialize the latest Vite in a new directory, call it whatever_you_like and scaffold it using the vanilla javascript template. Then, change into that new directory, like so.

$  npm init vite@latest whatever_you_like -- --template vanilla
$  cd whatever_you_like

2. Install Development Dependencies

Vite has already created some files, and we’re about to add a bunch more. Make sure you’re at the top level of your project directory in the terminal and run the command below. It’ll take a little while, since you’re bringing in so many files.

$  npm install -D @11ty/eleventy tailwindcss postcss autoprefixer alpinejs

There’s a bunch of other things you could bring in if you wanted, like plugins for the libraries you just installed such as @alpinejs/intersect for example, or additional libs, like html-minifier, but you can also install them later. For now, just stay focused.

3. Set up 11ty Infrastructure

Now is a good time to lay some basic groundwork. First, remove a few placeholder files (to be replaced later) and then create your directories. Go ahead and create a bunch of new files, while you’re at it.

$  rm index.html style.css main.js
$  mkdir _site src dist src/_data src/_includes src/images src/public src/styles
$  touch vite.config.js .eleventy.js src/_includes/base.html src/styles/style.css src/index.md src/main.js

You can break these up into multiple commands, if you want to take them slow, or just run them altogether.

4. Initialize Tailwind CSS

Tailwind needs some additional configuration. You can do this by running the command below from within your project directory.

$  npx tailwindcss init -p

After it’s done, go ahead and take a look at both postcss.config.js as well as tailwind.config.js just to make sure they got created. You’ll needto make some adjustments to the latter one in a bit.

5. Inspect the rest of your file structure

Right now, the top level of your project directory should look like this:

.
├── .eleventy.js
├── .gitignore
├── _site
├── dist
├── favicon.svg
├── node_modules
├── package-lock.json
├── package.json
├── postcss.config.js
├── src
├── tailwind.config.js
└── vite.config.js

The _site and dist directories should be empty, but the src directory should be well-stocked. It should look like the below.

src
├── _data
├── _includes
│   └── base.html
├── images
├── index.md
├── main.js
├── public
└── styles
    └── style.css

6. Configure Tailwind

The next step is kind of tedious, but super important so make doubly sure you’re not making any careless typos. You’ll be writing, or in some cases overwriting, content in files you’ve made in previous steps, so make sure to add the right text to the right files :).

Start with tailwind.config.js.

/*-- tailwind.config.js --*/
module.exports = {
  content: ['./src/**/*.html', './src/**/*.md'],
  theme: {
    extend: {},
  },
  plugins: [],
}

7. Configure Vite

Add the following lines to vite.config.js

/*-- vite.config.js --*/
import { defineConfig } from 'vite';
const { resolve } = require('path');
export default defineConfig({
    root: '_site',
    build: {
        outDir: '../dist',
        rollupOptions: {
            input: {
                main: resolve(__dirname, '_site', 'index.html'),
            },
        },
        emptyOutDir: true,
    },
});

8. Configure Eleventy

Add the following lines to .eleventy.js.

module.exports = function (config) {
    config.setLiquidOptions({
        dynamicPartials: true,
    });

    // Static assets to pass through
    config.addPassthroughCopy('./src/images');
    config.addPassthroughCopy('./src/public');
    config.addPassthroughCopy('./src/styles');
    config.addPassthroughCopy('./src/main.js');

    return {
        dir: {
            input: 'src',
            output: '_site',
        },
        passthroughFileCopy: true,
        templateFormats: ['html', 'md', 'liquid'],
        htmlTemplateEngine: 'liquid',
        dataTemplateEngine: 'liquid',
        markdownTemplateEngine: 'liquid',
    };
};

9. Configure npm via package.json

Even though most of your package.json file is already written for you, you will need to make a few adjustments to your scripts so you can start and run things most easily. Make your scripts section look like this:

"scripts": {
  "compile": "npx @11ty/eleventy",
  "dev": "vite",
  "build": "vite build",
  "preview": "vite preview",
  "full": "npm run build && npm run preview"
},

10. Configure AlpineJS

Finally, you’ll need to wire up the javascript library inside src/main.js.

/*-- src/main.js --*/
import Alpine from 'alpinejs'
window.Alpine = Alpine
Alpine.start()

11. Add a liquid template

Let’s set up a basic template that the other pages will be built on. Open up src/_includes/base.html and add the following content.

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8" />
        <link rel="icon" type="image/svg+xml" href="favicon.svg" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <link rel="stylesheet" href="/styles/style.css" />
        <title>{{ title }}</title>
    </head>
    <body class="h-screen w-screen flex justify-center items-center">
        <div class="md:w-1/4">
          {{ content }}
          <h1 x-data="{ message: 'Hello from Alpine' }" x-text="message"></h1>
        </div>
    <script type="module" src="/main.js"></script>
    </body>
</html>

The double curly-braces you see here are liquid variables replaced by markdown content from other files. The element classes are some tailwind utilities so we can test to see if it’s working.

12. Populate the template

Now add some markdown that will populate the template. You can do this by writing into the index.md file at the root of the src directory.

<!-- src/index.md -->
---
layout: "base.html"
title: Hello from Markdown
---
# Just Markdown?

Checking to make sure if there is markdown here.

## A second heading

And to see what paragraph spacing looks like in this model.

<p class="bg-emerald-500">Also to see if tailwind works like this.</p>

13. Import the Tailwind styles

The last bit of sample content you need are the style imports. These go inside src/styles/style.css.

@tailwind base;
@tailwind components;
@tailwind utilities;

14. Last steps

The final steps are to compile the build and deploy. The first time you do this, you should run the commands one at a time to help debug errors, but in the future you can run the build and preview commands automatically in sequence by using npm run full.

$  npm run compile
$  npm run build
$  npm run preview

If you don’t get any errors, you should be able to visit localhost:5000 in your browser and see your page. If there is a green background and you see a hello from Alpine, then everything is working.

15. Last steps

When you want to add more to your pages, you can stop the server by typing ctl+c (or cmd+c on Windows). Then, edit whatever you want and, when you’re done, rebuild the pages and re-launch the server.

npm run compile
npm run full

16. That’s it!

Now you have a working 11ty blog, with Tailwind CSS and Alpine JS. May it serve you well.
